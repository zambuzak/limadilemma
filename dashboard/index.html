3. H/s per visitor — diminishing returns past ~100 H/s. Browser ceiling is structural.

randomx.js v2 — Softfloat Implementation
Developer l-m confirmed v1 was archived just before completing softfloat (directed IEEE 754 floating-point rounding) support. This is the core WASM limitation for RandomX. v2 with softfloat = expected 2-4x hashrate improvement over current 38.64 H/s peak.
🔬 In development — not publicly available. Contact l-m on Discord for timeline.
RandomX v2Softfloat fix~2-4x gain
opd-ai/webminer — workerName Patch + WASM Investigation
Best consent-first framework but two blockers: (1) no workerName config field — session attribution via wallet string workaround only. (2) WASM always falls back to JS even with correct COOP/COEP headers and crossOriginIsolated=true. GitHub issue opened May 2026.
⏳ Awaiting developer response. workerName can be patched independently.
RandomXworkerName patchWASM broken
randomx.js-shared — SharedArrayBuffer Cache
Opus analysis identified shared cache as the single biggest available optimization (+30-40%). Each of 6 workers currently rebuilds its own 256MB cache = 1.5GB duplicate memory pressure destroying L3 locality. randomx.js-shared package not published on npm — requires building from archived source.
🔧 Needs source build from archived repo. Requires clang + WASM toolchain.
